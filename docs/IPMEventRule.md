## Overview

The `execute` method in the IPMEventRule class is responsible for executing a specific rule that updates the attributes of a vertex based on certain event records. This method processes input from a payload, performs necessary validations and transformations, and then updates the properties of the vertex accordingly.

## Usage

To use this class, you would typically call the `execute` method by passing in an instance of `RuleRunner`, a `Map<String, Object>` that contains your rule's configuration details (such as event type), the output of your rules (`RuleOutput` object), and a `Consumer<Object>` which is optional for customizing the behavior.

```java
IPMEventRule rule = new IPMEventRule();
rule.execute(ruleRunner, payloadMap, ruleOutput, consumer);
```

### Parameters

- **ruleRunner**: An instance of RuleRunner that will be used to execute your rules. This allows you to configure which actions or behaviors are performed by the rule.
- **payload**: A `Map<String, Object>` containing configuration details for your rule. The key-value pairs here should match the structure required by your event record's payload format.
- **ruleOutput**: An instance of RuleOutput that contains the output generated by this rule. This can be used to return results or pass them back to another part of your application.
- **consumer**: Optional - A `Consumer<Object>` that allows for additional actions like logging, saving state, etc., before and after execution.

### Returns

`execute` method does not return anything explicitly since it is an overridden method from the parent class RuleRunner. It simply executes the logic specified in its implementation.

## Methods

### Method: execute(RuleRunner ruleRunner, Map<String, Object> payload, RuleOutput ruleOutput, Consumer<Object> consumer)

#### Description

This method performs all necessary operations to process a specific event record, including validating attributes and updating vertex types based on resource type information provided by the input map. The updated attributes are then set back into the original vertex object for further processing.

#### Parameters

- **ruleRunner**: An instance of RuleRunner that will be used to execute your rules.
- **payload**: A `Map<String, Object>` containing configuration details for your rule. This map is expected to match the event record's payload format and should include keys like "configRecord" which contains a ConfigEvent object.
- **ruleOutput**: An instance of RuleOutput that will contain output generated by this rule.
- **consumer**: Optional - A `Consumer<Object>` for any custom logic or behavior that might need execution before or after the main processing.

#### Returns

This method does not return anything as it is an overridden implementation from the parent class. It simply performs its operations on the input parameters without explicitly returning a value.
